# Effect-template capability reference

_Generated by `npm run capability:doc` from the typed capability registry. Do not edit by hand._

This is the single source of truth for what a card assignment can express. Each row lists the exact params, the canonical card-text clauses it **covers**, what it explicitly **excludes** (route those to a new primitive or defer), and a real curated example to copy.

Totals: **97** effect fns · **72** gates · **4** cost kinds · **28** timings · **6** durations · **7** keywords.

## How to author a card

1. Segment the text by bracket tags (`[On Play]`, `[When Attacking]`, `[Activate: Main]`, `[DON!! xN]`, `[Trigger]`, `[Counter]`…) into abilities.
2. For each ability pick a **timing**; put board preconditions in `gate`, `[DON!! xN]`/turn state in `condition`, and activation costs (`:` clause) in `cost` or as an optional leading function + `ifPrevious`.
3. Map each effect clause to a **fn** below. If a clause matches an `excludes` entry or nothing at all, curate the mappable part and leave a `// PARTIAL:`/`// NOTE:` for the rest.
