## Partial bucket — clustered by root cause

**Correction (2026-07-12, later pass):** the original version of this doc claimed no `trashSelf`/`trashThis` primitive existed. That was wrong — both are fully live (`trashThis` is an `AbilityCost`, paid in `abilityCost.ts`; `trashSelf` is an effect fn compiling to `{ op: 'trashCards', target: { sel: 'self' } }`), already used across ~50 cards, with their own test (`trashThisCost.test.ts`). The "20 trash-self cards" cluster was really 20 cards whose *second* ability (the trashThis-gated one) was simply never added to the assignment. Fixed 12 of them by hand, reusing only already-wired primitives:

**Closed:** OP03-043, OP10-082, OP12-046, OP13-102, OP15-026, OP15-032, OP15-083, OP15-085, OP16-098, ST22-002 (also fixed its unrelated `excludeSelfName` gap while in the file), ST24-002, ST27-002.

**Still open in this cluster, do NOT copy the trashThis pattern blindly:**
- **OP16-021** (Stage) — `trashThis` cost is hardcoded to `characterArea` in `abilityCost.ts` (both the validity check and the payment code only touch `characterArea`). Paying it from a Stage would either reject the activation or corrupt zone state. Needs `abilityCost.ts` generalized to the source's actual zone before this can close safely.
- **OP08-045** — needs `registerKoReplacementSelf` to support a "trash self and draw N" combo; that fn currently only has plain `trashSelf?: true` (no draw). The aura variant (`registerKoReplacementAura`) has `trashSelfAndDraw`, but that's for protecting *allies*, not a genuine self-replacement — wrong fn for this card.
- **OP15-100** — the trash-self *and* Life-to-hand step is bundled as one optional compound cost inside an `onPlay` ability (not a separate `activateMain`); no precedent found for a multi-part optional cost like this outside the `cost` array mechanism. Needs a closer read before touching.
- **PRB02-009** — already has `trashThis` mapped (`onRested` → draw 2); the real gap is that `onRested` fires on *any* rest, not specifically "rested by opponent's effect." Needs a source-attribution condition that doesn't exist yet, not a missing ability.
- **ST13-007/010/014** — already have `trashThis` mapped; the real gap is the "reveal top Life, if it matches, may play it" conditional chain, approximated today with a different (weaker) sequence. Needs a proper reveal-then-conditional-play primitive, similar in spirit to the existing `revealTopThen` (deck-only) primitive but for Life.
